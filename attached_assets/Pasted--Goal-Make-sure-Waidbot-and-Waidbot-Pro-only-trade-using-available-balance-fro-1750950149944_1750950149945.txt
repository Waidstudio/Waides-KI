## ⚙️ Let’s Connect It

### 🔹 Step 1: Shared Wallet Context (React / JS)

Create a shared file so both wallet and bot access balance:

#### `src/context/SmaiWalletContext.js`

```jsx
import React, { createContext, useContext, useState } from 'react';

const SmaiWalletContext = createContext();

export const SmaiWalletProvider = ({ children }) => {
  const [smaiBalance, setSmaiBalance] = useState(1250); // example ₭ balance

  const updateBalance = (amount) => {
    setSmaiBalance(amount);
  };

  return (
    <SmaiWalletContext.Provider value={{ smaiBalance, updateBalance }}>
      {children}
    </SmaiWalletContext.Provider>
  );
};

export const useSmaiWallet = () => useContext(SmaiWalletContext);
```

---

### 🔹 Step 2: Update Wallet Page to Use Shared State

In your `SmaiSikaWallet.js`, change:

```js
import { useSmaiWallet } from '../context/SmaiWalletContext';

// Replace local state
const { smaiBalance, updateBalance } = useSmaiWallet();
```

Then use `updateBalance(newAmount)` when converting local to SmaiSika.

---

### 🔹 Step 3: Use Wallet in Waidbot Logic

Let’s say this is `Waidbot.js` or inside Waides KI engine:

```js
import { useSmaiWallet } from '../context/SmaiWalletContext';

export default function Waidbot() {
  const { smaiBalance } = useSmaiWallet();

  const startTrading = () => {
    if (smaiBalance < 10) {
      alert("Insufficient ₭ balance. Please top up your SmaiSika wallet.");
      return;
    }

    // Proceed with smart human-like trading logic
    console.log("Sufficient balance. Waidbot trading initiated...");
    // Trade logic here...
  };

  return (
    <div>
      <button onClick={startTrading}>🧠 Start Waidbot</button>
    </div>
  );
}
```

Same applies for Waidbot Pro. You can even show balance inside bot panel:

```jsx
<p>📊 Available Trading Power: ₭ {smaiBalance.toFixed(2)}</p>
```

---

### 🔐 Step 4: Prevent Over-Trading

In bot trade logic:

```js
if (tradeAmount > smaiBalance) {
  throw new Error("Trade exceeds available ₭");
}
```
